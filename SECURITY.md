# Security Policy

## Supported Versions

The IntelliFile project is actively maintained on the latest released version. Security fixes are generally applied to the current development branch and included in future releases.

| Version | Supported          |
| ------- | ------------------ |
| Latest release | ✅ |
| Older releases | ❌ |

## Reporting a Vulnerability

The IntelliFile project takes security issues seriously. If you discover a security vulnerability, please report it responsibly.

### Please Do Not

- Open a public GitHub issue for security vulnerabilities.
- Disclose vulnerability details publicly before a fix is available.

### How to Report

Please contact the maintainer directly:

- **Maintainer:** Stefan-Mihai MOGA
- **Repository:** [IntelliFile](https://github.com/mihaimoga/IntelliFile)

If contact information is available in the project documentation or maintainer profile, use those channels to submit your report.

### What to Include

To help assess and resolve the issue quickly, please include:

- A description of the vulnerability.
- Steps required to reproduce the issue.
- The affected IntelliFile version.
- Any proof-of-concept code, screenshots, or logs.
- Your assessment of the potential impact.

### Response Process

The project aims to:

1. Acknowledge receipt of the report within a reasonable timeframe.
2. Investigate and validate the vulnerability.
3. Develop and test an appropriate fix.
4. Release a patched version when necessary.
5. Credit the reporter (if desired) after public disclosure.

## Scope

Security reports are especially welcome for issues involving:

- Arbitrary code execution
- Privilege escalation
- File system access vulnerabilities
- Authentication or authorization weaknesses
- Unsafe handling of archives, FTP connections, or downloaded content
- Memory corruption, buffer overflows, or denial-of-service vulnerabilities
- Third-party dependency vulnerabilities that affect IntelliFile

## Responsible Disclosure

Please allow sufficient time for investigation and remediation before publicly disclosing any vulnerability details. Responsible disclosure helps protect users and the broader community.

Thank you for helping keep IntelliFile and its users safe.
